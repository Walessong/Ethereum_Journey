1. 项目全局认知

---------

本项目名为「以太坊思想史档案馆」，是一个高度聚焦于观点与冲突的单页面滚动叙事网站。请在所有代码生成过程中，严格参考 `PRD.md` 的叙事结构与 `TECH_DESIGN.md` 的技术架构。本项目的核心目标是通过代码展现技术路线的思想演进。

2. 核心开发规范

---------

* **类型安全**：强制使用 TypeScript。所有组件、外部传入的 props 以及存储历史冲突文本的数据结构，均需具备严格的接口定义（Interface）。

* **组件化原则**：采用 React 函数式组件配合 Hooks。

* **样式构建**：全面使用 Tailwind CSS 进行原子化样式编写。

* **动画引擎**：所有滚动视差与交互动画统一交由 Framer Motion 处理。
3. 代码风格与组织

----------

* **内容与逻辑解耦**：这是本项目的最高原则。严禁将大段具有文学或思想深度的文本硬编码在组件内部。所有文案数据必须存放于 `src/data/` 目录下的 JSON 文件中，由组件动态引入并渲染。

* **命名规范**：组件文件与函数名严格使用 PascalCase（如 `ConflictBlock.tsx`）。普通函数与变量使用 camelCase。

* **状态管理**：优先使用 React 原生 Context 或简单的状态提升，避免引入复杂的第三方状态管理库。
4. 视觉与排版指令 (The Vibe)

---------------------

* **极简克制**：UI 设计应偏向学术文献风格。保持大面积留白，通过字体字号的巨大反差来体现观念的冲突。

* **避免冗余控件**：无需设计花哨的按钮或边框。让文字内容本身成为界面的主体。

* **动效连贯**：在使用 Framer Motion 时，注重状态转换的平滑度。确保“信仰褪色”与“新叙事浮现”的过渡自然且不消耗过多性能。
5. 迭代与测试要求

----------

* 每次生成新组件后，需确保其在移动设备上的响应式排版依然能传达强烈的视觉张力。

* 重点关注滚动事件触发动画时的页面帧率。
